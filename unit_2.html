1. Introduction to Cryptography

What is Cryptography?

Cryptography is the practice of secure communication in the presence of adversaries. It uses mathematical algorithms to encrypt (encode) information so that only authorized parties can decrypt (decode) it.

Key Cryptographic Goals (CIA Triad)

üîê Confidentiality

Only authorized users can read the message. Encryption prevents unauthorized disclosure.

‚úì Integrity

Message hasn't been altered. Cryptographic hashes verify message authenticity.

üë§ Authentication

Verify sender's identity. Digital signatures prove the message source.

‚õî Non-Repudiation

Sender cannot deny sending message. Digital signatures provide proof.

2. Symmetric Encryption

Definition

Symmetric encryption uses the same key for both encryption and decryption. Both sender and receiver must possess the same secret key, which must be kept private.

How Symmetric Encryption Works

ALICE (Sender)
Plaintext + Secret Key
      ‚Üì (Encrypt)
Ciphertext
      ‚Üì (Send through unsecure channel)
BOB (Receiver)
Ciphertext + Secret Key
      ‚Üì (Decrypt)
Plaintext

Advantages

  • Fast encryption and decryption
  • Requires smaller keys than asymmetric
  • Simple and efficient implementation
  • Suitable for large data volumes
  • Well-established algorithms

Disadvantages

  • Key distribution problem (how to securely share key?)
  • Keys must be kept secret on both ends
  • Each pair of users needs unique key
  • Vulnerability if key is compromised
  • Scalability issues in large networks

Types of Symmetric Encryption

Stream Ciphers

  • Encrypt one bit/byte at a time
  • Uses keystream generator
  • Fast, minimal buffering needed
  • Examples: RC4, ChaCha20

Block Ciphers

  • Encrypt fixed-size blocks (64-256 bits)
  • Same plaintext block ‚Üí same ciphertext block
  • Uses modes (ECB, CBC, CTR, GCM)
  • Examples: AES, DES, 3DES

Key Management Challenge

The Problem

How do Alice and Bob securely exchange the secret key over an insecure channel?

Solutions

  • Diffie-Hellman Key Exchange
  • Key Distribution Center (KDC)
  • In-person key exchange
  • Combination with asymmetric encryption

3. Asymmetric Encryption

Definition

Asymmetric encryption (public-key cryptography) uses two mathematically related keys: a public key (known to everyone) and a private key (kept secret). Data encrypted with public key can only be decrypted with private key, and vice versa.

How Asymmetric Encryption Works

ALICE (Sender)
Plaintext + BOB's Public Key
      ‚Üì (Encrypt)
Ciphertext
      ‚Üì (Send - anyone can intercept)
BOB (Receiver)
Ciphertext + BOB's Private Key
      ‚Üì (Decrypt)
Plaintext

Advantages

  • Solves key distribution problem
  • Public key can be freely shared
  • Private key never needs to be transmitted
  • Enables digital signatures for authentication
  • Scales well to large networks
  • One key pair per user (not per user pair)

Disadvantages

  • Slower than symmetric encryption (50-1000x)
  • Requires much larger keys (2048+ bits)
  • More computationally intensive
  • Not practical for encrypting large volumes
  • Vulnerable to side-channel attacks if poorly implemented

Key Pair Generation

Property Public Key Private Key

Protection No protection needed Must use trusted storage (HSM)

Hybrid Encryption (Best of Both Worlds)

Combines Symmetric and Asymmetric

  1. Generate random symmetric key
  2. Encrypt large message with symmetric key (fast)
  3. Encrypt symmetric key with recipient's public key
  4. Send both encrypted message and encrypted key
  5. Recipient decrypts key with private key, then decrypts message

Used in: HTTPS, PGP, Signal messaging, TLS

5. RSA Algorithm

Overview

RSA (Rivest‚ÄìShamir‚ÄìAdleman) is the first practical asymmetric cryptography algorithm. It's based on the difficulty of factoring large numbers and is widely used for key exchange, digital signatures, and encryption.

RSA Specifications

Property Details
Invented 1977 (Rivest, Shamir, Adleman)
Key Sizes 512 to 4096 bits (recommended: 2048+)
Security Basis Factorization of large primes (hard problem)
Speed Slow (50-1000x slower than AES)
Use Case Key exchange, digital signatures, encryption (small data)

RSA Key Generation

  1. Choose Two Primes: Select large random primes p and q
  2. Calculate Modulus: n = p √ó q
  3. Calculate Phi: œÜ(n) = (p-1) √ó (q-1)
  4. Choose Public Exponent: e (typically 65537)
  5. Calculate Private Exponent: d = e‚Åª¬π mod œÜ(n)
  6. Public Key: (n, e)
  7. Private Key: (n, d)

RSA Encryption/Decryption

ENCRYPTION:
C = M^e mod n
(M = message, e = public exponent, n = modulus)

DECRYPTION:
M = C^d mod n
(C = ciphertext, d = private exponent, n = modulus)

RSA Digital Signatures

Signing

  • Hash message: h = Hash(message)
  • Sign with private key: s = h^d mod n
  • Send message + signature

Verification

  • Receive message + signature
  • Hash message: h' = Hash(message)
  • Verify: h' == (s^e mod n)
  • If equal, signature is valid and authentic

RSA Applications

  • HTTPS/TLS handshake (key exchange)
  • Email encryption and signing (PGP/GPG)
  • SSH key-based authentication
  • Code signing and verification
  • Certificate authorities (X.509 certificates)
  • Cryptocurrency key pair generation

RSA Security

Strengths

  • Well-studied mathematical foundation
  • No known efficient algorithm to factor large numbers
  • Quantum-resistant for 2048+ bits currently

Weaknesses

  • Vulnerable to quantum computers (Shor's algorithm)
  • Slow compared to symmetric encryption
  • Limited data size (max = key size - padding)
  • Vulnerable to padding oracle attacks if poorly implemented

Overview

DES (Data Encryption Standard) was the primary encryption standard before AES. Though now deprecated, it's historically important and still used in legacy systems and 3DES variant.

Why DES is No Longer Secure

  • Small Key: 56-bit key can be brute-forced in days/hours
  • Small Block: 64-bit blocks cause birthday attacks
  • Weak S-Boxes: Some non-linear operations are weak
  • Known Attacks: Meet-in-the-middle, differential cryptanalysis

3DES (Triple DES)

Solution to DES Weakness

Apply DES three times with different keys:

Ciphertext = DES_encrypt(DES_decrypt(DES_encrypt(Plaintext, K1), K2), K3)

Key Variants:
- 3DES with 3 keys: 168-bit effective security
- 3DES with 2 keys: 112-bit effective security

Still in use but deprecated: Slower than AES, only 112-bit security.

Historical Significance

  • First standardized encryption algorithm
  • Used to secure banking and government communications for 30+ years
  • Influenced design of all modern block ciphers
  • Demonstrated importance of key length (56 bits inadequate)
  • Led to development of AES as replacement

Real-World Applications

üåê HTTPS/TLS

Uses: RSA (key exchange), AES/ChaCha20 (data), SHA-256 (integrity)

Secures all web communication. Every website uses TLS 1.2+.

üí≥ Banking & Finance

Uses: AES (data), RSA (key exchange), HMAC-SHA256 (authentication)

ATMs, wire transfers, online banking all use cryptography.

üìß Email Security

Uses: PGP/GPG (RSA + AES), Digital signatures (RSA)

S/MIME and OpenPGP enable encrypted email communication.

üîê Password Storage

Uses: Hash functions (bcrypt, scrypt, Argon2)

Modern systems never store plaintext passwords, only hashes.

üí∞ Cryptocurrency

Uses: ECDSA (signing), SHA-256 (hashing)

Bitcoin and Ethereum use elliptic curve cryptography.

üîí File Encryption

Uses: AES-256 (BitLocker, FileVault, VeraCrypt)

Protect sensitive files and entire disk volumes.

Encryption in Different Scenarios

Scenario Encryption Type Algorithm Key Characteristics

File Storage Symmetric AES-256-CBC Fast, suitable for large files

Digital Signatures Asymmetric RSA, ECDSA Authentication, non-repudiation

Modern Cryptographic Practices

Recommended Algorithms

  • Symmetric: AES-256 (any mode except ECB)
  • Asymmetric: RSA-2048+ or ECDP-256+
  • Hashing: SHA-256, SHA-512, or Blake2
  • Key Exchange: ECDH or post-quantum alternatives
