How AES Encryption Works (Step by Step)

Step 1: Input Message

You provide the plaintext message you want to encrypt.

Example: "Hello World"

Step 2: Secret Key

You provide a secret key that only you and the recipient know.

Example: "MySecret"

Step 3: Divide into Blocks

The message is divided into 16-byte blocks (AES block size).

Message: "Hello World" Block Size: 16 bytes Result: ["Hello World "] (padded to 16 bytes)
Step 4: Key Expansion

The secret key is expanded into multiple round keys using AES key schedule.

Master Key ‚Üí Round Key 1 ‚Üí Round Key 2 ‚Üí ... ‚Üí Round Key 10

Step 5: Encryption Rounds (10 Rounds for AES-128)

Each block goes through 10 rounds of transformation:

  • SubBytes: Each byte is replaced using S-box lookup table
  • ShiftRows: Bytes are rearranged within the block
  • MixColumns: Columns are mixed using matrix operations
  • AddRoundKey: Block is XORed with round key

This process repeats 10 times, making the output completely scrambled.

Step 6: Ciphertext Output

The result is an encrypted message (ciphertext) that looks like random data.

Original: "Hello World" (readable)

Encrypted: "aj3k@#$%^&*()" (unreadable)

Decryption Process (Reverse)

Decryption is the reverse of encryption:

  1. Take the encrypted message
  2. Use the same secret key to generate round keys
  3. Apply inverse transformations in reverse order (10 rounds)
  4. Get back the original plaintext

Important: Without the correct key, decryption produces garbage data, not the original message!

Why AES is Secure

  • 256-Bit Key: 2^256 possible keys (more than atoms in universe)
  • Brute Force Resistant: Impossible to try all keys in reasonable time
  • Mathematical Strength: No known mathematical weakness
  • Confusion & Diffusion: Each output bit depends on all input bits and all key bits
  • Multiple Rounds: 10-14 rounds make pattern analysis impossible
  • Industry Standard: Used by governments, banks, and tech companies worldwide

  • Strong Keys: Use long, random keys (16+ characters) for better security
